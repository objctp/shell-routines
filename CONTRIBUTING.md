# Contributing to Shell Routines

This covers the non-obvious parts of working on the plugin. For what it does and how to install it, see the [README](README.md). For bug reports and pull requests, use the templates under `.github/`.

## Enable the git hook first

```bash
git config core.hooksPath .githooks
```

A `post-commit` hook regenerates the changelog's Unreleased section and folds it into the commit that just finished. Fresh clones don't have `core.hooksPath` set, so run this once or the automation below silently does nothing.

## Commits drive the changelog

CHANGELOG.md is generated by [git-cliff](https://git-cliff.org) from **conventional commits** â€” never hand-edit the top (Unreleased) section; it's regenerated.

- Type â†’ section: `feat`â†’Added, `fix`â†’Fixed, `perf`/`refactor`/`revert`/`docs`/`style`â†’Changed.
- `chore:` (and `docs(changelog):`) are **excluded** from the changelog. Use `chore:` for build, config, and housekeeping you don't want users to see.
- After each commit, the hook rewrites Unreleased and amends it into that same commit. It skips the release commit via `SHROUTINES_CHANGELOG_HOOK=1` (set only by the release script â€” don't set it for normal commits).
- The hook needs `git-cliff` on PATH; cutting a release also needs `jq`. Without `git-cliff`, the hook no-ops silently.

## Project layout and build

- `skills/` and `commands/` at the repo root are canonical. `.opencode/skills/` and `.opencode/commands/` are symlinks to them for OpenCode discovery.
- Agents are maintained per platform: `agents/*.md` (Claude Code frontmatter) and `.opencode/agents/*.md` (OpenCode format). Update both when changing an agent.
- `.claude-plugin/plugin.json` is the manifest Claude Code loads â€” update it when adding or removing an agent or command.
- `npm run build` runs `scripts/build.mjs`, which produces `dist/` (the npm package) from the sources above. Edit the sources, then rebuild â€” never hand-edit `dist/`.

## Tests, lint, format

- Shell tests run on **bashunit** under `tests/` (e.g. `tests/test-hooks.sh`). The `/shell-test-run` skill runs the suite with coverage enforcement.
- Shell is linted with **shellcheck** and formatted with **shfmt** (`-i 2`, two-space indent). TS/JS uses **Biome** (`biome.json`); spelling uses **cspell** (`cspell.json`); `bash-language-server` provides LSP via `.lsp.json` and `opencode.json`.

## Cutting a release

`scripts/changelog.sh` manages the whole thing:

- `preview` â€” print the regenerated Unreleased section.
- `unreleased` â€” rewrite Unreleased in CHANGELOG.md.
- `release [x.y.z]` â€” bump the version (auto-derived from commits if omitted), sync `package.json` and `.claude-plugin/plugin.json`, commit `chore(release): <version>`, and tag with `v`-prefixed semver (`v1.2.0`).

Then `git push --follow-tags` and create the GitHub release â€” that triggers `.github/workflows/publish.yml`, which publishes to GitHub Packages and npmjs.org. Don't `npm publish` locally: the repo `.npmrc` scopes `@objctp` to GitHub Packages and would send the publish to the wrong registry.
